  • 图片上传
    这里使用了picgo+markdown+gitee图床作为博客编写工具。

    自己结合一下四位大佬的代码，yy出了一个适合自己的插件。

    上传时间戳重命名前缀

    Typora+PicGo-Core+时间戳重命名 | Connor (connor-sun.github.io)

    【Picgo】正确重命名文件，避免重复上传 | 慕雪的寒舍 (musnow.top)

    gitee上传插件

    按照路径管理图片，方便上传和迁移

    当前插件功能

    • 上传前重命名文件(时间+hash)
    • 自动修改上传路径(以文件形式管理图片资源)
    • gitee插件魔改，当修改路径之后可以返回正确的链接
    • 如果gitee图床有同样的文件，不进行上传，转而返回本地数据库缓存链接

    优化点

    • 当前重复文件返回之后本地数据库中还是会有相同的链接，考虑优化

好用命令

  • git

    git submodule add https://gitee.com/xxx/xxx.git source

    可以将需要保护的文件设为私密仓，以子仓的形式管理

    git rm -r –cached lib/dw

    删除本地git 缓存

    git push origin HEAD:develop

    推送分支到远程develop分支

  • hexo

    hexo clean

    清除本地中间文件

    hexo s

    本地浏览器预览 hexo 界面

    hexo d -g –message “time@[commit info]”

    部署到git的时候添加commit信息

    hexo new draft b

    创建草稿

    hexo server –draft

    渲染草稿

    hexo publish b

    发布草稿

  • npm

    npm install hexo-renderer-pug hexo-renderer-stylus –save

    butterfly主题依赖插件

    npm install hexo-deployer-git –save
